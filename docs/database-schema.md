# База даних Balabony — опис схеми

Платформа використовує **Supabase** (PostgreSQL). Нижче описані ключові таблиці.

---

## Таблиця `plans` — тарифні плани

Довідник тарифів, доступних для придбання. Заповнюється вручну або через міграції.

| Колонка        | Тип         | Опис                                              |
|----------------|-------------|---------------------------------------------------|
| `id`           | TEXT (PK)   | Ідентифікатор плану: `monthly`, `annual`, `lifetime` |
| `name`         | TEXT        | Відображувана назва: «Місячна підписка» тощо      |
| `price_kopecks`| INT         | Ціна в копійках (грн × 100). 9900 = 99 грн        |
| `interval`     | TEXT        | Інтервал повтору: `month`, `year` або NULL         |
| `is_recurring` | BOOLEAN     | `true` — автоматичне поновлення через LiqPay       |
| `is_active`    | BOOLEAN     | `false` — план прихований від покупки              |
| `created_at`   | TIMESTAMPTZ | Час створення запису                               |

### Поточні тарифи

| id         | Назва              | Ціна      | Тип           |
|------------|--------------------|-----------|---------------|
| `monthly`  | Місячна підписка   | 99 грн    | Авторекурентна |
| `annual`   | Річна підписка     | 799 грн   | Авторекурентна |
| `lifetime` | Довічний доступ    | 2 999 грн | Одноразова     |

---

## Таблиця `subscriptions` — підписки

Зберігає активні та минулі підписки користувачів. Ідентифікація — за `device_id`.

### Нові поля (міграція 005)

| Колонка               | Тип         | Опис                                                      |
|-----------------------|-------------|-----------------------------------------------------------|
| `email`               | TEXT        | Email для надсилання квитанцій (необов'язково)            |
| `plan_id`             | TEXT (FK)   | Посилання на `plans.id` — який тариф придбано             |
| `is_first_month`      | BOOLEAN     | Перший розрахунковий період (знижка або тріал)            |
| `liqpay_token`        | TEXT        | Токен LiqPay для автоматичного повторного списання        |
| `next_charge_at`      | TIMESTAMPTZ | Дата наступного автосписання                              |
| `next_charge_kopecks` | INT         | Сума наступного списання в копійках                       |
| `cancelled_at`        | TIMESTAMPTZ | Дата скасування підписки; NULL = підписка активна         |

---

## Таблиця `purchases` — поштучні покупки

Записи про покупку окремих історій без підписки.

| Колонка             | Тип         | Опис                                                        |
|---------------------|-------------|-------------------------------------------------------------|
| `id`                | UUID (PK)   | Унікальний ідентифікатор покупки                            |
| `device_id`         | TEXT        | Ідентифікатор пристрою покупця                              |
| `email`             | TEXT        | Email покупця (необов'язково)                               |
| `story_id`          | UUID (FK)   | Посилання на `stories.id` (без CASCADE — запис зберігається після видалення історії) |
| `amount_kopecks`    | INT         | Сума оплати в копійках                                      |
| `liqpay_order_id`   | TEXT        | ID замовлення в системі LiqPay                              |
| `liqpay_payment_id` | TEXT        | ID платежу в системі LiqPay                                 |
| `status`            | TEXT        | Стан: `pending` / `success` / `failed` / `refunded`        |
| `created_at`        | TIMESTAMPTZ | Час створення запису                                        |

---

## Таблиця `stories` — аудіоісторії

Основна таблиця контенту платформи.

| Колонка             | Тип         | Опис                                                        |
|---------------------|-------------|-------------------------------------------------------------|
| `id`                | UUID (PK)   | Унікальний ідентифікатор                                    |
| `title`             | TEXT        | Назва історії                                               |
| `author_name`       | TEXT        | Ім'я автора                                                 |
| `genre`             | TEXT        | Жанр (пригоди, казка, фантастика тощо)                      |
| `text`              | TEXT        | Оригінальний текст, поданий автором                         |
| `corrected_text`    | TEXT        | Текст після редакторської правки Claude                     |
| `humanized_text`    | TEXT        | «Природніший» варіант тексту від Claude                     |
| `published_version` | TEXT        | Яка версія публікується: `original` / `corrected` / `humanized` / `corrected_humanized` |
| `cover_url`         | TEXT        | URL обкладинки                                              |
| `status`            | TEXT        | `pending` / `approved` / `rejected`                         |
| `approved_at`       | TIMESTAMPTZ | Час затвердження адміном                                    |

---

## Row Level Security (RLS)

| Таблиця         | Стан RLS | Політики                                          |
|-----------------|----------|---------------------------------------------------|
| `plans`         | Увімкнено | `plans_select_all` — читання для всіх (anon + authenticated) |
| `purchases`     | Увімкнено | Без конкретних політик — буде додано при підключенні клієнта |
| `subscriptions` | Не чіпали | Наявні політики збережено без змін                |
